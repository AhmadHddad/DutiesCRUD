# API Reference

Base local URL: `http://localhost:4000`

Interactive Swagger UI: `/docs`

Raw OpenAPI spec: `/openapi.json`

## Conventions

- Successful responses use `{ "data": ... }`.
- Errors use `{ "error": { code, message, requestId, details? } }`.
- `X-Request-Id` is accepted from callers or generated by the API and returned in responses.
- `ETag` is returned on single-duty read and update responses.
- `If-Match` is required for updates.

## Error Envelope

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Duty name is required.",
    "requestId": "f7d8..."
  }
}
```

`details` is included when the error needs structured context, such as the latest duty after a stale edit.

## `GET /health`

Returns backend and database health.

### Success Response

```json
{
  "status": "ok",
  "database": "ok",
  "timestamp": "2026-05-29T12:00:00.000Z"
}
```

### Failure

- `503 SERVICE_UNAVAILABLE` when the database check fails.

## `GET /api/duties`

Lists duties with offset pagination.

### Query Parameters

- `limit`: optional integer, defaults to `50`, minimum `1`, maximum `100`.
- `offset`: optional integer, defaults to `0`, minimum `0`.

### Important Behavior

- If `limit` or `offset` is repeated, the first value is used.
- Invalid integers or out-of-range values return `400 VALIDATION_ERROR`.

### Success Response

```json
{
  "data": {
    "items": [
      { "id": "1", "name": "Plan release" }
    ],
    "total": 2,
    "limit": 50,
    "offset": 0,
    "nextOffset": null
  }
}
```

## `GET /api/duties/:id`

Returns one duty by id.

### Path Parameters

- `id`: positive integer string.

### Success Response

Headers:

- `ETag: "duty-<id>-v<version>"`

Body:

```json
{
  "data": {
    "id": "1",
    "name": "Plan release"
  }
}
```

### Failures

- `400 VALIDATION_ERROR` when the id is not a positive integer string.
- `404 NOT_FOUND` when the duty does not exist.

## `POST /api/duties`

Creates a new duty.

### Request Body

```json
{
  "name": "Prepare sprint review"
}
```

### Validation Rules

- `name` must be a string.
- surrounding whitespace is trimmed.
- the trimmed value must not be empty.
- the trimmed value must be `256` characters or fewer.
- names are treated as plain text and are not HTML-sanitized.
- unexpected body fields are ignored.

### Success Response

```json
{
  "data": {
    "id": "1",
    "name": "Prepare sprint review"
  }
}
```

### Failures

- `400 VALIDATION_ERROR`
- `429 TOO_MANY_REQUESTS` when the write rate limit is exceeded.

## `PUT /api/duties/:id`

Updates a duty.

### Path Parameters

- `id`: positive integer string.

### Required Header

- `If-Match: "duty-<id>-v<version>"`

### Request Body

```json
{
  "name": "Updated duty name"
}
```

### Success Response

Headers:

- `ETag: "duty-<id>-v<newVersion>"`

Body:

```json
{
  "data": {
    "id": "1",
    "name": "Updated duty name"
  }
}
```

### Failures

- `400 VALIDATION_ERROR` for invalid ids, invalid body data, or malformed `If-Match`.
- `404 NOT_FOUND` when the duty does not exist.
- `428 PRECONDITION_REQUIRED` when `If-Match` is missing.
- `412 PRECONDITION_FAILED` when the duty changed since the editor loaded it.
- `429 TOO_MANY_REQUESTS` when the write rate limit is exceeded.

### Stale Update Response

```json
{
  "error": {
    "code": "PRECONDITION_FAILED",
    "message": "Duty has changed since you opened it. The latest duty has been loaded.",
    "requestId": "f7d8...",
    "details": {
      "latestDuty": {
        "id": "1",
        "name": "Current server name"
      }
    }
  }
}
```

The response also includes the latest `ETag` header so the frontend can retry with fresh state.

## `DELETE /api/duties/:id`

Deletes a duty.

### Success Response

- Status `204 No Content`

### Failures

- `400 VALIDATION_ERROR` for invalid ids.
- `404 NOT_FOUND` when the duty does not exist.
- `429 TOO_MANY_REQUESTS` when the write rate limit is exceeded.

## Headers And Cross-Origin Notes

The backend CORS layer allows:

- methods: `GET, POST, PUT, DELETE, OPTIONS`
- request headers: `Content-Type`, `X-Request-Id`, `If-Match`
- exposed response headers: `X-Request-Id`, `ETag`
